What is "DotCloud"

What is "DotCloud"

"Be a developer, not a sysadmin."

What is "DotCloud"

  • PaaS (Platform as a Service) のひとつ
  • 同じ系のサービス
    • Google App Engine (Java,Python,Go)
    • Heroku (Ruby,Node.js)

What is "DotCloud"

  • Perl (PSGI) が使えるのが画期的
  • 対応言語たくさん
    • Perl/PHP/Ruby/Python/Node.js/Java ...
  • データベースもたくさん
    • MySQL/PostgreSQL/MongoDB/CouchDB ...
  • 現在はベータサービス

DotCloudでの Perlアプリケーション実行環境

  • Webサーバ: nginx
  • アプリケーションサーバ: uwsgi

サインアップ (ベータ登録)

みなさん、やってますよね？

"dotcloud" コマンドのインストール

# python
$ sudo easy_install dotcloud

"dotcloud" コマンド(1) - help

$ dotcloud -h
# サブコマンドヘルプ
$ dotcloud deploy -h

"dotcloud" コマンド(2) - create

namespace を作成
namespace の中には複数の service (アプリケーション) が登録できる 

$ dotcloud create dragon3

[ アプリケーションのURL ]

http://[service].[namespace].dotcloud.com/

"dotcloud" コマンド(3) - list

namespace のリストと
その namespace にぶら下がっている service のリストを表示 

$ dotcloud list
dragon3:
  - dragon3.www     (perl) 
  - dragon3.nopaste (perl) 
  - dragon3.mojo1   (perl)

"dotcloud" コマンド(4) - deploy

新しいサービスを登録 (配備環境作成)

  • namespace : dragon3
  • service : hello
  • 言語 : perl

$ dotcloud deploy -t perl dragon3.hello

"dotcloud" コマンド(5) - push

アプリケーションの配備(アップロード) 

$ dotcloud push dragon3.hello ~/ws/hello/

"dotcloud" コマンド(5) - push のまえに (1)

最初に作ったhelloアプリケーションを配備します。 

.
├── Makefile.PL
├── app.psgi
└── hello.pl

"dotcloud" コマンド(5) - push のまえに

dotcloud に配備できるのは PSGI仕様に準拠したアプリ
Mojolicious であれば、、、 

$ emacs app.psgi
require 'hello.pl';

"dotcloud" コマンド(5) - push のまえに

サーバ側に依存ライブラリを準備するために Makefile.PL を作成 

$ emacs Makefile.PL
use ExtUtils::MakeMaker;
WriteMakefile( PREREQ_PM => { 'Mojolicious' => '1.42' } );

アプリケーションURLにアクセスしてみる
http://hello.dragon3.dotcloud.com/

"dotcloud" コマンド(5) - push

$ dotcloud push dragon3.book .

アプリケーションURLにアクセスしてみる
http://hello.dragon3.dotcloud.com/

"dotcloud" コマンド(6) - logs

アクセスログやエラーログを表示(tail -f) 

$ dotcloud logs dragon3.book1

"dotcloud" コマンド(7) - ssh

サーバにSSH 

$ dotcloud ssh dragon3.book1

"dotcloud" コマンド(7) - その他

ヘルプで見てね 

$ dotcloud -h
